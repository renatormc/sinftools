s-report3

A nova versão do programa s-report inclui as seguintes alterações:

  1. Estruturado para processamento e análise de vários objetos e várias fontes de dados ao mesmo tempo
  2. Suporte para todos os tipos de arquivo não mais somente a áudio, imagem e vídeo.
  3. Analisador mais simples com versão portável para poder ser enviado para o requisitante.

Para utilizar siga os passos a seguir

1-Criar a estrutura de pastas

Crie uma pasta de trabalho e dentro dela crie uma pasta para cada objeto do caso.

Dentro da pasta de cada objetos coloque uma pasta para cada fonte de dados relacionados aquele objeto

No exemplo acima a pasta "teste_report" é a pasta de trabalho. As pastas C2, C3 e C4 são as pastas relacionadas aos objetos no caso são 3 celulares. No celular C2 e C3 as fontes de dados são apenas relatório do ufed. Já no caso do celular C4 além do relatório do UFED foram feitas outras extrações sqlite e spi.

obs: Para o programa funcionar é necessário organizar os dados nessa estrutura. Os nomes das pastas você pode escolher de acordo com suas preferências mas a estrutura deve ser mantida.

2- Preparar o ambiente

Abra o terminal  na pasta de trabalho e digite "s-report3  init".

3- Editar os arquivos de configuração

Edite os arquivos de configuração de extensão "yaml" que foram gerados um em cada pasta de fonte de dados

4- Processar os dados

Digite no terminal "s-report3 process" para ler e processar os dados.

5- Analisar dados (opcional)

Dê dois cliques no arquivo 'sinf_analyzer.exe' para abrir o analisador e faça os filtros e marcações de desejados

6- Gerar relatório

Você pode optar em enviar para o requisitante relatório em html ou enviar o próprio analisador.

Para gerar relatório html:

Abra o arquivo ".report\report_config.yaml" e configure o seu relatório. Em seguida digite "s-report3 render" para gerar o relatório em html

Para enviar o analisador:

Digite o comando "s-report3 portable". Isso fará com que os arquivos necessários sejam copiados e um executável seja criado na pasta de trabalho.

 

 Caso não se lembre dos comandos necessários digite "s-report3 --help" para vizualizar os comandos existentes.

 

Tipos de fontes de dados

1- Relatório XML do UFED

A pasta da fonte de dados nesse caso é a pasta que contém o arquivo xml

2- SPI Tools

É o aplicativo desenvolvido pelos períto de São Paulo que exporta as conversas para uma pasta no formato texto e os anexos. Para utilizar esta fonte você precisa cirar a pasta fonte e dentro dela criar duas pastas "anexos" e "chats". Dentro da pasta "anexos" devem ser colocados os arquivos de anexo e dentro da pasta "chats" os arquivos de textos gerados pela ferramenta spi_tools.


3- Sqlite

Você precisa ter conseguido extrair de alguma forma os arquivos msgstore.db, wa.db e a pasta de anexos do WhatsApp. Então deve colocar tais arquivos dentro da pasta da fonte de dados. Coloque os arquivos na pasta fonte conforme a figura a seguir, sendo Media a pasta que contém os anexos.

4- Arquivos na pasta

Este tipo de extração e para quando tiver uma pasta cheia de arquivos e subpastas e queira incluir no relatório da extração. Para isso basta colocar a pasta que contém os arquivos como fonte de dados.
